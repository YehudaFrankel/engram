Why it's different

Three tiers. Most tools ship one.

Memory is the foundation. But memory alone doesn't learn from mistakes or act without prompting. Claude Recall ships all three — and they compound.

🔁
The living system advantage

Other memory tools are static — you document once, things go stale. Claude Recall stays alive: memory syncs to git after every session, skills patch themselves from failure data, and each session builds directly on the last. It's not a snapshot. It's a system that grows.

TIER 01
Memory
Claude remembers your codebase, decisions, patterns, and known bugs across every session. Syncs to git — pull on any machine and pick up where you left off. Nothing is re-explained.
TIER 02
Skills
Skills score themselves after every use. /evolve reads the scores and rewrites the failing step automatically. The same skill failure never happens twice — without you touching anything.
TIER 03
Autonomous
Skills chain into multi-step workflows. Self-healing retries before escalating. Memory auto-pushes after 9pm. Drift detection catches undocumented changes after every edit. Claude acts.

📝
/learn — extract what happened
Writes to lessons.md, decisions.md, and skill_scores.md. Scores every skill that fired: Y if it needed correction, N if it worked first time. The "What Failed" column is what /evolve reads to patch the right step.
📊
skill_scores.md — binary log
A running table of every skill, every session. /evolve finds rows where Correction Needed = Y and Improvement Applied = — and fixes them. Specific failure data → specific patch.
⚡
/evolve — patch and cluster
Rewrites the failing step in the skill file. Clusters repeated lessons across sessions into new reusable skills. Logs every change to skill_improvements.md. The same mistake is never made twice.
🔁
Next session — compounded
Patched skills load at Start Session. Lessons apply before any code is touched. Rejected approaches in regret.md are never re-proposed. Each session is smarter than the last.

Autonomous Behaviors

It runs. You work.

No commands, no checkpoints — these happen automatically throughout every session.

⛓ Skill Chaining

Add one section to any skill and it automatically triggers the next — pass or fail. No human steps between.

## Auto-Chain
On pass: → verification-loop
On fail: → debug-topic, retry

fix-bug → verify → smoke-test
↓ fail
debug → smoke-test
🔧 Self-Healing

When a verify step fails, Claude attempts the minimal fix and retries once before escalating to you. Define the fix in a Recovery section.

## Recovery
1. touch the changed file
2. wait 5s for reload
3. re-run smoke test
4. escalate if still fails

✓ auto-recovery succeeded
🌙 Auto End Session

After 9pm with unsaved memory changes, the stop hook auto-commits and pushes to git. Memory is never lost — even if you forget End Session.
